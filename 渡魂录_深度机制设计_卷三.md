### 1.1 阮如月 · 体系 A：悲惧压制流（主套）

#### 体系定义

**核心资源**：悲印/惧印层数
**主要循环**：施悲惧印 → 积累至共鸣阈值 → 触发爆发效果 → 情绪牌补充手牌 → 再施印
**爆发窗口**：悲共鸣（下次伤害×2）叠加惧共鸣（跳过行动）→ 同回合打出高伤牌
**防御逻辑**：惧印跳过敌人行动 = 主动防御；封印之眼提供被动护盾
**渡化适配**：★★★★★（悲惧情绪是第一卷主要情绪，印记直接转渡化进度）
**镇压适配**：★★★（印记共鸣可造成大量伤害，但清场速度不如沈铁钧）

#### 内部资源流转图

```
[情绪牌：悲歌 0费]
    → 悲+1 → 抽1张牌（悲慧被动）
    → 手牌增加 → 打出更多施印牌
    → 悲印层数+2（烙印）
    → 悲印×3 → 悲共鸣：下次受伤×2
    → 打出攻击牌触发悲共鸣翻倍
    → 惧印×3 → 惧共鸣：跳过行动
    → 安全输出回合 → 继续施印
```

#### 核心牌型角色分工

| 牌型 | 代表牌 | 在循环中的角色 |
|------|--------|--------------|
| 引擎牌 | 悲歌（0费） | 0费驱动整个循环 |
| 施印主力 | 烙印✦（悲×4） | 快速堆悲印至共鸣 |
| 控制支点 | 魂缚✦（惧×5） | 直接触发惧共鸣+行动控制 |
| 爆发清算 | 悲浸✦（悲印×5=20伤） | 将印记转化为输出 |
| 终结器 | 五蕴共鸣✦ | 一次清算所有共鸣 |
| 渡化节点 | 渡印诀✦ | 将印记换治疗+进度 |

#### 弃牌在此体系中的专属价值

- 弃掉「悲歌」：产生**悲情绪碎片×2**（悲歌本身加悲+1，弃掉额外加）
- 弃掉施印牌：产生**对应情绪碎片×1**
- 弃掉「五蕴共鸣」：产生**五色碎片×1**（稀有，用于锻造）
- **弃牌触发「印散」效果**：每弃1张牌，随机对场上一个敌人施加随机印记×1（悲慧·弃牌变体遗物激活后）

#### 节奏弱点与对策

| 弱点 | 表现 | 对策 |
|------|------|------|
| 起手无印记 | 第1-2回合几乎无输出 | 走魂册遗物（起手按施印牌数量自动施印） |
| 印记被清除 | 精英技能清除所有印记，积累归零 | 印记之源（恢复被消耗印记）/ 碎镜片遗物 |
| 高费牌堆积 | 五蕴共鸣2费，冥印术2费，手牌卡住 | 庙祝法衣遗物（共鸣后下一张牌-1费） |
| 渡化条件不满足 | 情绪值不足 | 情染牌（+2并施印联动） |

### 1.2 阮如月 · 体系 B：光愈流（变体）

#### 体系定义

**核心资源**：喜印层数 + 「花开」状态
**主要循环**：喜祷施喜印 → 花开触发（喜共鸣：下一张牌免费）→ 0费打出愈灵诀/喜溢 → 回血+再施印
**爆发窗口**：定印×3触发定共鸣（本回合全部牌-1费）× 花开状态 = 近乎无限手牌输出回合
**防御逻辑**：高频回血 + 定印共鸣降费产生更多防御行动次数
**渡化适配**：★★★★（喜情绪对应第三卷，光与愈直接推进度）
**镇压适配**：★★（治疗型打法输出较低，依赖定印共鸣的降费打出更多伤害牌）

#### 内部资源流转图

```
[喜祷 1费]
    → 喜印×2 + 回复5HP
    → 定澄 0费 → 定+1 + 定印×2
    → 空灵护体 1费 → 护盾（定×4）+ 喜+1
    → 花开 1费（喜印≥2）→ 触发喜共鸣 + 「花开」状态
    → 「花开」：下一张牌免费
    → 免费打出愈灵诀（回复HP=喜印+定印总层数）
    → 或免费打出双印汇流（同时施喜印×2+定印×2，触发双共鸣）
    → 循环：喜印→花开→免费回血→再喜印
```

#### 花开状态的精确规则

「花开」是光愈流最核心的机制，规则如下：

```
触发来源：
  - 花开（牌卡）：打出后当回合下一张牌免费
  - 镜花（工具牌）：复制上一次共鸣，如果上次是喜共鸣则再给一次花开
  - 无为（终极技）：持续3-5回合，每回合开始自动给一次花开

花开状态规则：
  - 「花开」同一回合可叠加（叠2次 = 接下来2张牌都免费）
  - 「花开」不跨回合（回合结束清零）
  - 叠加上限：5次（防止无限循环）

花开触发的双重收益：
  - 正常：下一张牌费用归0
  - 配合定印共鸣（全牌-1费）：效果叠加，本来1费的牌变0费，0费的牌仍0费
```

#### 核心牌型角色分工

| 牌型 | 代表牌 | 在循环中的角色 |
|------|--------|--------------|
| 引擎牌 | 喜泉（0费×3/回合） | 0费堆喜情绪+喜印种子 |
| 施印核心 | 喜祷（施印+治疗双效） | 主力施喜印，同时回血 |
| 花开触发 | 花开（条件触发） | 引爆「免费」连锁 |
| 定印支持 | 定澄（0费定印） | 0费推进定印，辅助定共鸣 |
| 治疗核心 | 愈灵诀（总印记数=HP） | 主要治疗来源 |
| 渡化收口 | 光与愈（+进度+大量HP） | 渡化路线的终结节点 |

#### 弃牌在此体系中的专属价值

- 弃掉喜情绪相关牌：产生**喜情绪碎片**，碎片可在锻造台兑换「花开Token」（额外一次花开状态）
- 弃掉「镜花」：因为此牌有2张，有时过多，弃掉触发「镜影」：当回合最后一次共鸣效果翻倍
- 弃牌数量≥3时：愈灵诀回复量+50%（「空手换血」弃牌遗物激活后）

#### A体系 vs B体系对比

| 维度 | 悲惧压制流（A） | 光愈流（B） |
|------|--------------|-----------|
| 首选情绪 | 悲、惧 | 喜、定 |
| 主要输出来源 | 悲浸/惧噬/共鸣伤害 | 定印共鸣后的连打伤害牌 |
| 生存方式 | 惧共鸣跳过行动（主动） | 高频回血（被动持续） |
| 弃牌收益 | 情绪碎片→再施印 | 花开Token→更多免费牌 |
| 最佳关卡 | 第一卷（悲惧主场） | 第三卷（喜定主场） |
| 渡化速度 | 快（悲印直接+进度） | 中（喜印需要积累后光与愈） |
| 操作复杂度 | ★★★（印记类型管理） | ★★★★（花开叠加计数） |

### 1.3 沈铁钧 · 体系 A：锁链怒爆流（主套）

#### 体系定义

**核心资源**：锁链层数 + 怒情绪值
**主要循环**：施锁链 → 用怒涌推怒情绪 → 怒至3-4触发怒爆 → 怒爆期间打高伤牌引爆溅射 → 击杀后锁链继承 → 继续
**爆发窗口**：怒=4触发怒爆（攻击×1.5）→ 怒煞（6+4×4=22伤）→ 锁链溅射（×5层×42%=约9点AOE） → 整场大量伤害
**防御逻辑**：铁壁（护盾10+定+1）+ 怒盾（护盾随怒增长）；定情绪作为防御辅助
**渡化适配**：★★★（需要走定情绪辅助渡化，怒爆流本身倾向镇压）
**镇压适配**：★★★★★（锁链溅射是游戏内效率最高的清场机制）

#### 内部资源流转图

```
[铁链掷✦ 1费]
    → 施锁链×2 + 造成11伤
    → 锁链层数2（溅射26%）
    
[怒涌✦ 0费]
    → 怒+2 + 下次攻击+7伤

[再打铁链掷]
    → 怒现在=2，锁链=4（溅射42%）

[怒涌再次]
    → 怒=4：触发怒爆（×1.5攻击）

[怒煞✦ 1费]
    → 8+5×4=28点伤 × 1.5（怒爆）= 42点伤
    → 锁链溅射：42×42%=17点AOE

[震链✦ 1费]
    → 对所有锁链敌人9伤 × 1.5 = 13.5点 × 锁链溅射
    
[回合结束：怒归零]
    → 下回合：老手直觉遗物锁链继承，怒气壶+1怒，继续积累
```

#### 怒情绪精确管理

怒爆流的核心操作是**怒情绪的精确推进**，避免被迫进入失调而不是主动触发怒爆：

```
怒情绪状态机：
  怒=0-2：积累期（打怒涌，推进锁链）
  怒=3：准备期（此时下一个怒+1就触发怒爆，最佳时机打怒放：怒+3直接触发怒爆）
  怒=4：怒爆期（本回合攻击×1.5，清算所有高伤牌）
  怒爆结束：怒归0，重新积累

关键判断：
  - 打怒放（怒+3）：最快从怒=1进入怒爆，但消耗1费
  - 打怒涌×2（各+2）：从怒=0需要3张，但都是0费
  - 不要在怒=3时打怒+2的牌（怒会到5，触发失调而不是怒爆！）
```

#### 锁链溅射价值计算表

理解溅射效率有助于决策：

| 锁链层数 | 溅射% | 怒=4时溅射% | 定≥3时额外 | 最终溅射% |
|---------|------|-----------|----------|---------|
| 1层 | 18% | 38% | +20% | 58% |
| 2层 | 26% | 46% | +20% | 66% |
| 3层 | 34% | 54% | +20% | 74% |
| 4层 | 42% | 62% | +20% | 82% |
| 5层 | 50% | 70% | +20% | 90% |

> 5层锁链 + 怒爆 + 定≥3 = 溅射90%，等于每次攻击都有90%的伤害AOE扩散。

#### 弃牌在此体系中的专属价值

- 弃掉怒相关牌：产生**怒情绪碎片**，可在锻造台消耗碎片「强化怒爆」（怒爆倍率临时+0.1，本场有效）
- 弃掉施锁牌：锁链不消失，弃掉时随机将1层锁链转移给当前敌人（即使不打出也有部分施锁效果）
- 弃掉「怒爆诀」：特殊——弃掉时对所有锁链敌人造成（当前怒情绪×3）点穿甲伤害（「余怒」机制）

### 1.4 沈铁钧 · 体系 B：铁壁流（变体）

#### 体系定义

**核心资源**：铁甲值 + 定情绪值
**主要循环**：堆定情绪 → 铁壁/定锁获得高护盾 → 铁甲覆体（护盾40%转铁甲）→ 护盾残余继续积累 → 定而无极（铁甲转攻击力）
**爆发窗口**：铁甲积累≥15后，定而无极✦（铁甲×6=攻击力加成，本回合有效）→ 铁壁突刺✦（护盾×45%=输出）→ 以柔克刚✦（铁甲×4=输出）
**防御逻辑**：铁甲是固定减伤，不依赖每回合护盾积累；万夫莫开（护盾减少时补回35%）实现永久护盾
**渡化适配**：★★（定情绪高，可辅助渡化，但打法本身不面向渡化）
**镇压适配**：★★★★（持久战碾压，Boss战极强）

#### 铁甲的精确运作

铁甲是铁壁流的专属机制，规则需要精确定义：

```gdscript
# 铁甲规则（GDScript实现参考）
var armor_value: int = 0  # 铁甲值，固定减伤
var armor_cap: int = 30   # 铁甲上限

# 受到伤害时的计算顺序
func take_damage(raw_damage: int) -> void:
    # 1. 护盾先吸收
    var after_shield = max(0, raw_damage - shield)
    shield = max(0, shield - raw_damage)
    
    # 2. 铁甲减伤（固定值，不消耗铁甲本身）
    var after_armor = max(0, after_shield - armor_value)
    
    # 3. 剩余伤害扣HP
    hp -= after_armor
    
    # 4. 万夫莫开遗物：护盾被消耗时补回35%
    if shield_was_reduced:
        shield += int(shield_reduction * 0.35)
```

#### 铁甲积累路径

| 方法 | 铁甲获得量 | 前提条件 |
|------|-----------|---------|
| 铁甲覆体（牌） | 护盾×40% | 需要先有护盾 |
| 铁甲覆体✦ | 护盾×55% | 升级后更高效 |
| 百炼成钢（牌） | 本回合受伤总量×1 | 主动挨打换铁甲 |
| 百炼成钢✦ | 本回合受伤总量×1.5 | 升级后效率更高 |
| 无动于衷（回合积累） | 间接：定不衰减 → 护盾持续高 → 更多可转铁甲 | 被动积累 |
| 钢铁意志✦ | 本回合总受伤×100% | 受伤即转护盾再转铁甲 |

#### 铁甲值 vs 护盾值的战略选择

```
铁甲优先：长期战斗，Boss战，敌人高频攻击
  优点：永久存在，固定减伤不受攻击次数影响
  缺点：需要时间积累，早期脆弱

护盾优先：短期战斗，普通敌人，爆发清场
  优点：当回合立即生效，量大
  缺点：回合结束清零（除非铁身/静若磐石）

最优策略（铁壁流中期）：
  护盾 → 铁甲覆体（55%转铁甲）→ 剩余45%护盾继续防御本回合
  → 下回合护盾重新积累 → 铁甲持续增加
  → 20-30轮后铁甲≥25，每次受伤几乎无实际HP损失
```

#### 弃牌在此体系中的专属价值

- 弃掉防御牌：产生**定情绪碎片**，碎片可强化铁甲上限（+5，本局）
- 弃掉「百炼成钢」：弃掉时获得护盾等于当前铁甲值×20%（以甲换盾的弃牌反向操作）
- 弃牌触发「钢铁余韵」：弃掉任意牌，铁甲+1（执念之铁遗物激活后）

#### A体系 vs B体系对比（沈铁钧）

| 维度 | 锁链怒爆流（A） | 铁壁流（B） |
|------|--------------|-----------|
| 核心资源 | 锁链层数+怒情绪 | 铁甲值+定情绪 |
| 爆发模式 | 怒爆瞬间爆发（单回合最高） | 铁甲积累后持续高压 |
| 多敌效率 | ★★★★★（溅射天然AOE） | ★★★（铁链回旋）|
| Boss效率 | ★★★★（天网恢恢）| ★★★★★（铁甲抗压极强）|
| 容错率 | 低（怒情绪管理失误损失大） | 高（铁甲永久存在，失误代价低）|
| 弃牌收益 | 怒碎片→怒爆强化 | 定碎片→铁甲上限扩展 |

### 1.5 无名（无面人）· 体系 A：空鸣爆发流

#### 体系定义

**核心资源**：空度（0-10）+ 空鸣触发次数
**主要循环**：精确将空度从低推至高（或反向）→ 经历全部分段 → 触发空鸣 → 空度归零 → 下一轮重新积累
**爆发窗口**：空度=9-10（所有牌0费）→ 打出所有手牌（连续多张伤害牌）→ 空鸣（全敌25点穿甲+渡化+25%）
**防御逻辑**：空度越低（0-2）受伤越多但效果越强；空度越高（6-10）虽然HP-5/回合但牌费降低可打更多防御
**渡化适配**：★★★★（空鸣直接+25%进度，渡口令再+15%，配合情绪转移推进度）
**镇压适配**：★★★★（空鸣25点穿甲AOE，空中劈情绪爆发）

#### 空鸣触发的三条路径

```
路径A：自然经历（低风险，低频）
  每回合打牌空度+1 → 约6-8回合自然到10 → 回合开始-2 → 难以精确触发
  → 适合新手，但很少真正触发空鸣

路径B：主动控制（中风险，中频）  
  空收（空度-3）→ 虚实互换（互换当前空度）→ 空溢（空度+3）
  → 在2-3回合内完成0→10的全程经历
  → 适合大多数中期场景

路径C：强制引爆（高费，高效）
  空鸣诀（3费）→ 直接触发空鸣，空度归零
  空鸣诀✦（2费）→ 效果还+50%
  → Boss战或紧急情况下的终结技
```

#### 空度分段的战术价值

```
空度0-2（低段）：用于主动输出
  - 所有效果+10%
  - 打虚无掌（5+空度×2，空度=1时=7×1.1=7.7）
  - 打借力打力（敌情绪×5穿甲）
  - 风险：容错低，受伤多

空度3-5（中段）：用于过渡操作
  - 无加成无惩罚，最安全
  - 打情绪转移牌（借情、情债）
  - 打工具牌（观形、一念）
  - 此时最适合弃牌操作（弃牌不会浪费分段收益）

空度6-8（高段）：用于广度攻势
  - 每张牌费用-1
  - 打大量牌（6费的牌变3费，可以打双倍数量）
  - 打铺场性质的牌（情镜、借情×多个目标）
  - 风险：HP不可回复

空度9-10（极高段）：用于终结回合
  - 所有牌0费（但每回合HP-5）
  - 在这一段打出所有手牌
  - 使用一念（弃全手牌+1抽同等+1）刷手牌
  - 下一回合触发空鸣
```

#### 弃牌在此体系中的专属价值

- 弃掉任意牌：空度+1（主动调整空度的辅助手段）
- 弃掉情绪转移牌：不产生碎片，而是立即转移目标最高情绪×1给无面人（即使没打出也有部分效果）
- 弃掉「一念」：特殊——弃掉时空度先+（弃掉前手牌数），再-5（回调），产生一次小幅空度波动

### 1.6 无名（无面人）· 体系 B：情绪操控流

#### 体系定义

**核心资源**：敌方情绪值 + 「情绪负债」状态
**主要循环**：情渡（将自身情绪转移给敌人）→ 敌人情绪积累至失调 → 失调惩罚触发 → 借力打力（敌方情绪×5穿甲）→ 情绪炸裂（一次触发多个失调效果）
**爆发窗口**：敌方有2+种情绪值≥3时 → 情绪炸裂（每个失调效果触发+14伤）→ 移情别恋（五情互换，可能瞬间引发多种失调）
**防御逻辑**：情债（目标情绪增加无效）阻止敌人技能效果 + 镜面护（受伤转情绪给敌人）
**渡化适配**：★★★★★（情渡将自身情绪转给敌人，然后渡化那个情绪，完全不需要积累自身情绪）
**镇压适配**：★★★（情绪爆炸伤害可观，但需要铺垫）

#### 情绪操控流的关键裁判：「情绪负债」

「情绪负债」是此体系的核心控制状态：

```
情绪负债（Emotion Debt）状态：
  来源：情债（牌）
  效果：目标本回合行动后，本回合获得的所有情绪值清零
  
实际作用：
  - 阻止敌人通过行动积累情绪（破坏敌人自身的情绪累积策略）
  - 阻止敌人Buff中的情绪增加效果（让治愈类敌人无法恢复）
  - 配合移情别恋：先用情绪负债阻止敌人下回合积累，再用移情别恋把敌人当前情绪值互换
```

#### 体系 A vs 体系 B 对比（无名）

| 维度 | 空鸣爆发流（A） | 情绪操控流（B） |
|------|--------------|--------------|
| 核心操作 | 精确管理自身空度 | 操控敌方情绪状态 |
| 爆发节奏 | 每3-6回合一次大爆发 | 持续稳定压力 |
| 操作复杂度 | ★★★★（空度计算） | ★★★★★（情绪状态预判） |
| 对多敌效率 | ★★★（空鸣AOE） | ★★★★（情镜扩散情绪） |
| 对Boss效率 | ★★★★（空鸣穿甲） | ★★★★★（移情别恋瞬间引发多失调） |
| 弃牌收益 | 空度调整工具 | 情绪转移部分效果 |

## 第二章：弃牌机制（Discard System）

> 弃牌不是丢掉手牌，而是将手牌转化为另一种资源的选择。
> 每次弃牌都有意义，每个角色从弃牌中获得不同的收益。

---

### 2.1 弃牌的触发时机

弃牌分三种来源：

| 来源 | 触发方式 | 说明 |
|------|---------|------|
| 主动弃牌 | 玩家点击「弃牌」按钮，最多弃1张/回合（除非遗物/牌效改变） | 主动决策 |
| 被动弃牌 | 惧情绪失调效果（回合开始随机弃1张）/ 特定敌人技能 | 被迫弃牌 |
| 效果弃牌 | 打出某些牌时要求弃掉其他牌（如「一念」）| 主动触发的被动弃牌 |

**弃牌的统一基础规则**：
- 被弃的牌进入弃牌堆（默认），而非移除
- 弃牌时必然触发「情绪碎片」产出（见2.2）
- 弃牌不消耗能量

---

### 2.2 情绪碎片系统（Emotion Shard）

情绪碎片是弃牌机制的核心输出资源，连接弃牌与锻造系统。

#### 碎片产出规则

```
弃牌时，根据被弃牌的「情绪标签」产出对应碎片：

情绪类牌（悲/惧/怒/喜/定 标签）：
  → 对应情绪碎片 ×2

施印/施锁/空度操控牌：
  → 对应机制碎片 ×1（印记碎片/锁链碎片/空度碎片）

工具/防御/攻击牌（无情绪标签）：
  → 灵气碎片 ×1（通用）

稀有情况：
  弃掉升级过的牌（✦标记）：产出碎片 ×2
  被迫弃牌（失调/敌技能）：产出碎片 ×1（降低，代表情绪失控的浪费）
  弃掉「传说级」遗物关联牌：产出「残响碎片」×1（极稀有，锻造最高级材料）
```

#### 碎片类型与用途

| 碎片类型 | 来源 | 用途 |
|---------|------|------|
| 悲情绪碎片 | 弃悲相关牌 | 锻造「悲愿」系牌 / 强化悲印效果 |
| 惧情绪碎片 | 弃惧相关牌 | 锻造「惧囚」系牌 / 强化惧印效果 |
| 怒情绪碎片 | 弃怒相关牌 | 锻造「怒煞」系牌 / 临时+0.1怒爆倍率 |
| 喜情绪碎片 | 弃喜相关牌 | 锻造「喜愈」系牌 / 额外花开Token |
| 定情绪碎片 | 弃定相关牌 | 锻造「定甲」系牌 / +5铁甲上限 |
| 印记碎片 | 弃施印牌 | 锻造高级印记组合牌 |
| 锁链碎片 | 弃施锁牌 | 锻造高级锁链牌 |
| 空度碎片 | 弃空度操控牌 | 锻造空鸣强化牌 |
| 灵气碎片 | 弃通用牌 | 锻造通用改造 |
| 残响碎片 | 弃升级牌/特殊触发 | 锻造角色专属极品改造 |

#### 碎片存储与展示

```
UI 展示位置：
  战斗中：顶部 HUD 右侧，遗物栏旁，显示当前积累的碎片数量和类型图标
  战斗外：地图/商店界面左下角，「碎片袋」图标，点击展开详情

存储上限：
  每种碎片上限 20 个（防止无限积累破坏平衡）
  碎片跨战斗保留（一局游戏内持续积累）
  在锻造台消耗，或在特殊事件中使用

碎片的「腐化」机制：
  若某种碎片积累到上限（20个），超出部分不产出，但触发「碎片共鸣」：
  该情绪的牌本场战斗效果+15%（过剩碎片转化为当场增益）
```

---

### 2.3 各角色弃牌专属机制

除了碎片产出，每个角色弃牌时有独特的附加效果：

#### 阮如月：弃牌即「印散」

每当阮如月主动弃牌时（非被动失调弃牌），触发「印散」：
- 随机对场上一个敌人施加「碎片印记×1」（类型 = 被弃牌的情绪标签对应的印记类型）
- 若无情绪标签，施加随机印记×1

效果意味着：**弃牌不只是换碎片，而是同时推进了印记积累**。

**升级后（遗物「残香」激活）**：印散效果变为施加碎片印记×2，且触发「悲慧」（情绪增加时抽牌）。

#### 沈铁钧：弃牌即「余怒」

每当沈铁钧主动弃牌时，触发「余怒」：
- 若被弃的牌有「怒」标签：对当前锁链目标造成（怒情绪值×2）点伤害
- 若被弃的牌是施锁牌：对随机敌人施加锁链×1（即使没打出也有施锁效果）
- 若被弃的牌是防御牌：定+1

效果意味着：**沈铁钧的手牌「好牌打，次牌弃」都有收益**。

#### 无名（无面人）：弃牌即「空流」

每当无名主动弃牌时，触发「空流」：
- 空度+1（弃牌本身就是调整空度的工具）
- 若当前空度因此进入新分段（如从5变为6，或从2变为3），立即获得该分段的「进入奖励」：
  - 进入0-2段：回复3HP
  - 进入3-5段：抽1张牌
  - 进入6-8段：获得能量+1
  - 进入9-10段：下一张牌免费

效果意味着：**无名的弃牌是精确控制空度分段的工具，不是浪费，是操作**。

### 2.4 弃牌牌池（Discard Synergy Cards）

以下牌卡专门与弃牌机制协同，从商店/事件中获得：

---

**留不住的** | 费用0 | 通用
弃掉1张手牌（玩家选择），抽2张牌，产出被弃牌的碎片×3（而非正常×2）。
*设计意图*：主动丢差牌换手牌质量，同时获得额外碎片。

---

**身后事** | 费用1 | 通用
将弃牌堆顶的3张牌选其中1张加入手牌，其余2张产出碎片（正常量）。
*设计意图*：从弃牌堆中回收价值，弃牌堆成为手牌储备库。

---

**清空** | 费用0 | 通用
弃掉手中所有牌（整手弃掉），每弃1张产出灵气碎片×2，下回合手牌上限+3（本场有效）。
*升级*：每弃1张产出对应类型碎片×2（而非灵气），下回合手牌上限+5。

---

**执念最后** | 费用2 | 阮如月专属
弃掉手中所有印记相关牌，每弃1张对目标施加对应印记×2。然后抽等同弃掉数量的牌。
*设计意图*：一次性将大量手牌转化为印记爆发，同时补充手牌。

---

**怒去无踪** | 费用0 | 沈铁钧专属
弃掉所有手中「怒」标签的牌，每弃1张：对所有锁链敌人造成（当前怒情绪值×2）伤害。怒情绪归零。
*设计意图*：将怒情绪的「余量」一次性清算，适合怒爆后的清理。

---

**虚空归处** | 费用1 | 无名专属
弃掉手中空度最高的那张牌（标注各牌的空度消耗），空度精确变化为5（中段）。抽2张牌。
*设计意图*：快速归中的工具，在极端空度下强制回到安全区间。

## 第三章：升级系统（Upgrade System）

> 升级不只是数值提升，而是**打开牌的第二层用法**。
> 每张牌的升级方向决定了它在体系中的角色转变。

---

### 3.1 升级渠道

| 渠道 | 获得方式 | 条件 |
|------|---------|------|
| 休息点升级 | RestScene 选择「升级一张牌」 | 到达休息点 |
| 渡化奖励 | 渡化成功后额外1次免费升级 | 阮如月渡化成功 |
| 锻造升级 | 在锻造台消耗材料升级 | 需要特定碎片 |
| 特定事件 | 部分随机事件提供升级机会 | 概率触发 |
| 传说遗物 | 观尘之镜：每3场战斗自动升级1张随机牌 | 持有遗物 |

---

### 3.2 升级五型分类（详细规则）

#### 强化型（Power Upgrade）

**规则**：核心数值提升约50%。适用于需要「更多就是更好」的数值型牌。

```
计算公式：
  基础伤害 N → 升级后 N + max(1, floor(N × 0.5))
  基础护盾 N → 升级后 N + max(2, floor(N × 0.5))
  印记层数 N → 升级后 N + 1（印记不按50%，防止过强）
  回复量 N → 升级后 N + max(1, floor(N × 0.5))

示例：
  烙印（悲印×2）→ 烙印✦（悲印×3）
  铁壁（护盾10）→ 铁壁✦（护盾14，而非15，取整）
  怒煞（6+怒×4）→ 怒煞✦（8+怒×5）
```

#### 费用型（Cost Upgrade）

**规则**：费用-1（最低0）。适用于频繁打出的循环牌。

```
费用0的牌不可进行费用型升级（已经最低）
费用3的牌费用型升级后变费用2（仍然不是免费）

视觉识别：
  费用角标变为金色填充圆形（普通是暗色）

例外：若费用型升级会使一张"次回合才生效"的牌变为0费，
该牌需要加入限制"每场战斗只能打出1次"防止无限循环
```

#### 扩展型（Extend Upgrade）

**规则**：增加新效果行（通常是范围扩展或附加效果）。这是设计空间最大的升级类型。

```
扩展方向：
  A. 单体→全体（铁链掷：对一个目标 → 对一个目标+对最近敌人造成2点伤害）
  B. 范围扩大（观形：揭示1个 → 揭示所有敌人）
  C. 附加小效果（烙印✦：施印后额外抽1张牌）
  D. 跨回合效果（束缚诀✦：翻倍效果延长至下回合）

扩展型不改变原有效果，只增加，保持可预测性
```

#### 解锁型（Unlock Upgrade）

**规则**：移除牌卡上的限制条件。适用于有明确使用门槛的牌。

```
限制类型和解锁方式：
  「只能对锁链目标」→ 移除，可对任意目标
  「目标印记总层数≥X」→ 条件降低（≥8→≥6）
  「每场战斗只能打出1次」→ 次数+1（1次→2次）
  「本回合最后一张牌」→ 移除，变为普通打出
  「只能在受到伤害后」→ 移除，变为随时可打出

视觉：限制文字变灰色并加横线（表示条件已解锁删除）
```

#### 转化型（Transform Upgrade）

**规则**：效果质变，是五种升级中最强但也最稀少的。每套牌至多3张转化型升级牌。

```
转化的标准：
  - 升级前后是"完全不同的打法节点"
  - 不能只是"变强了"，要是"变成了另一张牌的感觉"
  - 通常改变的是效果的触发方式或作用对象

现有转化型升级牌：
  余怒未消（沈铁钧）：升级前「本回合消耗怒×5伤」→ 升级后「当前怒×6伤（不消耗怒）」
  五印封体（阮如月）：单体施印 → 对所有敌人施印
  调息（无名）：精确设定任意值 → 只能设0或10（更极端）
```

### 3.4 升级的视觉表现系统

```
升级完成时的动画效果（BattleScene 和 RestScene 通用）：
  1. 牌卡轻微浮起（向上位移8px，0.2s）
  2. 金色光晕从中心向外扩散（0.3s，淡出）
  3. 牌名文字末尾「✦」符号逐字出现（0.15s）
  4. 升级标记（费用/描述/边框颜色）同步更新

视觉差异化标准：
  费用型：费用角标背景色变金色（#c8a020），数字白色
  强化型：描述中变化的数字高亮显示（变化前的值↗变化后的值，显示0.5s后固定为新值）
  扩展型：新增效果文字以浅金色渐入
  解锁型：原限制条件文字加删除线后渐出
  转化型：整个卡面颜色深化（主题色饱和度+30%），边框由单线变双线
```

## 第四章：锻造系统（Forge System）

> 锻造是游戏最深度的资源投入机制，将弃牌积累的碎片转化为**永久性的牌卡改变**。
> 锻造不是升级，而是「改造」——改变牌的形态甚至类型。

---

### 4.1 锻造台的出现

锻造台不是固定设施，而是通过以下方式解锁：

| 触发条件 | 获得方式 |
|---------|---------|
| 地图上的「锻造事件」节点 | 特定章节地图节点（每层地图0-1个） |
| 随机事件「流浪炉匠」 | 概率约15%的事件节点 |
| Boss奖励选项 | 第1-2章Boss战胜后，三选一奖励中可能出现「使用锻造台一次」 |
| 遗物「残香（七束）」 | 第7次共鸣奖励：获得「临时锻造台」（本场战斗外使用） |
| 商店特殊服务 | 商店花费100金购买一次锻造机会（稀有出现） |

---

### 4.2 锻造的核心操作

**锻造的基本流程**：

```
① 选择一张牌（从当前牌组中选）
② 选择锻造类型（见4.3）
③ 查看所需材料（碎片类型+数量）
④ 确认消耗，完成锻造
⑤ 牌卡永久改变（本局内）
```

**锻造与升级的区别**：

| 维度 | 升级 | 锻造 |
|------|------|------|
| 改变幅度 | 渐进式提升 | 质变式改造 |
| 改变类型 | 只强化原有效果 | 可改变牌的类型/机制 |
| 材料消耗 | 无（机会消耗） | 情绪碎片 |
| 可撤销 | 不可撤销 | 不可撤销 |
| 每张牌次数 | 1次 | 1次 |
| 是否叠加升级 | — | 可在已升级的牌上锻造 |

---

### 4.3 锻造类型全览（共8种）

---

#### 锻造·印 「情绪刻印」

**消耗**：对应情绪碎片 ×5 + 灵气碎片 ×2

**效果**：为一张牌添加「情绪标签」。此后该牌每次被打出，该情绪+1。

**可添加到**：任意无情绪标签的牌（攻击牌、防御牌、工具牌）

**例子**：
- 铁链掷（原无情绪标签）→ 刻印「怒」→ 铁链掷现在每次打出：造成8伤+施锁×1+怒+1
- 灵护（防御牌）→ 刻印「定」→ 灵护每次打出：护盾8+定+1

**叠加锻造**：若在已有情绪标签的牌上再次「情绪刻印」，改为：该牌的情绪效果加倍（情绪+2而非+1）。

---

#### 锻造·衔 「机制衔接」

**消耗**：印记碎片或锁链碎片 ×5（取决于目标机制）

**效果**：为一张普通攻击/防御牌添加「衔接效果」——当此牌被打出后，激活当前场上最近一次触发的印记/锁链效果的小型复现。

**衔接强度**：原效果的40%（数值取40%，层数按×0.4向上取整）

**例子**：
- 铁壁（护盾10）→ 锻造「机制衔接·印」→ 铁壁每次打出：护盾10 + 对目标施加当前场上最近触发的印记类型×1
- 怒煞（6+怒×4）→ 锻造「机制衔接·锁」→ 怒煞每次打出：造成伤害 + 对目标施加锁链×1

---

#### 锻造·倍 「共鸣倍增」

**消耗**：对应情绪碎片 ×8 + 残响碎片 ×1

**效果**：为一张「触发共鸣」类牌添加「倍增」属性：该牌触发的共鸣，效果数值×1.5（向上取整）。

**仅可锻造**：五蕴共鸣、印叠诀、悲浸、惧噬、共鸣回响

**叠加效果**：若该牌已经升级（✦），倍增在升级后效果基础上计算。

**例子**：
- 五蕴共鸣✦（触发所有共鸣×1.2）→ 锻造「共鸣倍增」→ 五蕴共鸣✦锻 = 触发所有共鸣×1.2×1.5=×1.8

---

#### 锻造·缚 「锁链强化」

**消耗**：锁链碎片 ×6 + 怒情绪碎片 ×3

**效果**：为一张「施锁」牌添加「强化锁链」属性：此牌施加的锁链，溅射伤害系数额外+5%（不计入每层8%的基础，而是固定附加）。

**仅可锻造**：施锁类牌（铁链掷、锁喉、锁阵、缚灵术、天网、重锁等）

**叠加**：可以在同一张牌上多次使用「锻造·缚」（每次需要重新消耗碎片），每次+5%，最多叠加3次（+15%）。

**例子**：
- 铁链掷✦（施锁×2）→ 锻造「锁链强化」×2 → 铁链掷✦锻 = 施锁×2，此锁链溅射额外+10%

---

#### 锻造·换 「情绪置换」

**消耗**：任意两种情绪碎片各 ×4

**效果**：将牌的情绪标签从A换为B（消耗哪两种碎片，A=前者，B=后者）。

**可置换范围**：改变牌的「情绪归类」标签，但**不改变**牌的数值效果。

**用途示例**：
- 悲歌（悲标签）→ 锻造「情绪置换：悲→喜」→ 喜歌（喜标签，但效果仍是：0费，+1情绪，抽1张牌）
- 打出「喜歌」的行为现在触发「悲慧」被动中「喜」情绪增加时抽牌，也触发「印堂朱砂」遗物中喜共鸣的联动

**战略意义**：
- 使非主情绪标签的牌转化为主情绪标签，扩大协同范围
- 将情绪标签统一到同一类型，触发遗物的「同类情绪集中效益」

---

#### 锻造·魂 「亡魂共鸣」

**消耗**：五色碎片 ×1（五情共鸣触发时偶尔产出，极稀有） + 灵气碎片 ×5

**效果**：为任意一张牌添加「魂触发」：此牌打出时，若场上有未完成渡化的印记，渡化进度+5%。

**累积效应**：若同一回合内打出了多张带「魂触发」的牌，渡化进度叠加（打4张×5%=+20%）。

**适用角色**：主要对阮如月有价值（沈铁钧和无名也可使用，但渡化进度对他们意义较小）。

---

#### 锻造·空 「空度注入」

**消耗**：空度碎片 ×5 + 灵气碎片 ×3（仅无名可使用）

**效果**：为无名的一张牌添加「空度注入」：此牌打出时，可选择「主动调整空度±1」（在+1或-1中选择一个）。

**战略价值**：使无名的每一次出牌都同时是一次精确的空度调整，将空度控制从「专用牌」扩展到「全体牌」。

**叠加**：若在已有「空度注入」的牌上再次锻造，调整范围扩展为±2（可选+2/-2/+1/-1中的任意一个）。

---

#### 锻造·极 「极限改造」（最高级）

**消耗**：残响碎片 ×2 + 对应角色专属碎片 ×10

**效果**：将一张牌「重铸」为同类型的更高级版本。具体效果由角色和牌型决定：

**阮如月可用**：
- 任意施印牌 → 「极印」版本：费用+1，但施印层数×3，且触发共鸣时不消耗层数（改为-1层）
- 任意触发牌 → 「极共鸣」版本：将所有条件去掉，但每场战斗只能打出1次

**沈铁钧可用**：
- 任意施锁牌 → 「极锁」版本：施加锁链上限去除（突破5层上限，最高可到8层），但费用+1
- 任意攻击牌 → 「极怒」版本：伤害额外增加（当前怒情绪×5），但打出后怒-2

**无名可用**：
- 任意空度操控牌 → 「极空」版本：调整幅度×2，且不产生空度副作用（不触发HP-5的风险分段）
- 任意情绪转移牌 → 「极转」版本：转移量翻倍，且同时对所有敌人生效（由单体变群体）

---

### 4.4 锻造图纸（Recipe System）

锻造图纸是解锁特殊锻造组合的方式，不是随机发现，而是通过游戏推进解锁：

| 全角色使用 | 「混炉·锻法」 | 使用过3种角色（多周目） |

#### 图纸的效果（示例）

解锁特殊锻造：「印化（花费：悲碎片×3+惧碎片×3）」
将一张攻击牌转化为「渡化攻击牌」：造成伤害时，同时推进渡化进度+10%。

**空鸣炉·锻法**（无名专属）

效果：情绪转移操作同时精确控制空度，双机制协同无需分开操作。

## 附录：GDScript 实现参考

### 弃牌系统核心代码

```gdscript
# DiscardSystem.gd — 建议集成到 DeckManager.gd 中
class_name DiscardSystem

signal shard_generated(shard_type: String, amount: int)
signal discard_special_effect(card_id: String, effect_type: String)

# 情绪碎片存储
var shards: Dictionary = {
    "bei": 0, "ju": 0, "nu": 0, "xi": 0, "ding": 0,
    "seal": 0, "chain": 0, "void": 0, "spirit": 0, "echo": 0
}
const SHARD_CAP = 20

func discard_card(card: Dictionary, is_forced: bool = false) -> void:
    # 1. 产出碎片
    var shard_amount = 1 if is_forced else 2
    if card.get("upgraded", false):
        shard_amount = int(shard_amount * 2)  # 升级牌双倍碎片
    
    var shard_type = _get_shard_type(card)
    _add_shard(shard_type, shard_amount)
    
    # 2. 触发角色专属弃牌效果
    _trigger_character_discard_effect(card, is_forced)
    
    # 3. 将牌加入弃牌堆
    DeckManager.add_to_discard(card)

func _get_shard_type(card: Dictionary) -> String:
    var tags: Array = card.get("tags", [])
    var emotion_map = {"悲":"bei","惧":"ju","怒":"nu","喜":"xi","定":"ding"}
    
    for emotion in emotion_map:
        if emotion in tags:
            return emotion_map[emotion]
    
    if "施印" in tags or "印记" in tags:
        return "seal"
    if "施锁" in tags or "锁链" in tags:
        return "chain"
    if "空度" in tags:
        return "void"
    return "spirit"  # 通用碎片

func _add_shard(type: String, amount: int) -> void:
    var prev = shards.get(type, 0)
    var new_val = prev + amount
    
    if new_val > SHARD_CAP:
        # 碎片溢出：触发碎片共鸣
        _trigger_shard_resonance(type)
        new_val = SHARD_CAP
    
    shards[type] = new_val
    emit_signal("shard_generated", type, amount)

func _trigger_character_discard_effect(card: Dictionary, is_forced: bool) -> void:
    if is_forced:
        return  # 被迫弃牌不触发专属效果
    
    match GameState.current_character:
        "ruanyueyue":  # 阮如月
            _ruanyueyue_discard(card)
        "shentiejun":  # 沈铁钧
            _shentiejun_discard(card)
        "wumian":      # 无名
            _wumian_discard(card)

func _ruanyueyue_discard(card: Dictionary) -> void:
    # 印散：随机对敌人施加对应印记×1
    var tags = card.get("tags", [])
    var emotion_to_seal = {"悲":"bei","惧":"ju","怒":"nu","喜":"xi","定":"ding"}
    var seal_type = "bei"  # 默认随机
    
    for e in emotion_to_seal:
        if e in tags:
            seal_type = emotion_to_seal[e]
            break
    
    var random_enemy = BattleStateMachine.get_random_alive_enemy()
    if random_enemy:
        SealManager.add_seal(random_enemy, seal_type, 1)
        emit_signal("discard_special_effect", card.get("id",""), "seal_scatter")

func _shentiejun_discard(card: Dictionary) -> void:
    # 余怒：根据牌的标签触发对应效果
    var tags = card.get("tags", [])
    var nu = EmotionManager.get_emotion("nu")
    
    if "怒" in tags and nu > 0:
        var target = BattleStateMachine.get_chained_target()
        if target:
            BattleStateMachine.deal_damage(target, nu * 2, false)
    elif "施锁" in tags or "锁链" in tags:
        var target = BattleStateMachine.get_random_alive_enemy()
        if target:
            ChainManager.add_chain(target, 1)
    elif "防御" in tags or "护盾" in tags:
        EmotionManager.add_emotion("ding", 1)

func _wumian_discard(card: Dictionary) -> void:
    # 空流：弃牌调整空度 + 分段进入奖励
    var prev_stage = _get_void_stage(WumianManager.void_level)
    WumianManager.add_void(1)
    var new_stage = _get_void_stage(WumianManager.void_level)
    
    if new_stage != prev_stage:
        _trigger_stage_enter_bonus(new_stage)

func _get_void_stage(level: int) -> int:
    if level <= 2: return 0
    if level <= 5: return 1
    if level <= 8: return 2
    return 3

func _trigger_stage_enter_bonus(stage: int) -> void:
    match stage:
        0: GameState.modify_hp(3)    # 进入低段：回复3HP
        1: DeckManager.draw_cards(1) # 进入中段：抽1张牌
        2: GameState.add_energy(1)   # 进入高段：+1能量
        3: DeckManager.set_next_card_free(true)  # 进入极高段：下一张牌免费
```

```gdscript
class_name ForgeSystem
extends Node

signal forge_completed(original_card: Dictionary, forged_card: Dictionary)
signal recipe_unlocked(recipe_id: String)

const FORGE_TYPES = {
    "emotion_brand":   {"name":"情绪刻印","shard_cost":{"emotion":5,"spirit":2}},
    "mechanism_link":  {"name":"机制衔接","shard_cost":{"seal_or_chain":5}},
    "resonance_amp":   {"name":"共鸣倍增","shard_cost":{"emotion":8,"echo":1}},
    "chain_enhance":   {"name":"锁链强化","shard_cost":{"chain":6,"nu":3}},
    "emotion_swap":    {"name":"情绪置换","shard_cost":{"emotion_a":4,"emotion_b":4}},

    "void_inject":     {"name":"空度注入","shard_cost":{"void":5,"spirit":3}},

func can_forge(card: Dictionary, forge_type: String) -> bool:
    if card.get("forged", false):
        return false  # 已锻造过（每张牌只能锻造1次）
    
    var cost = FORGE_TYPES[forge_type]["shard_cost"]
    return DiscardSystem.has_enough_shards(cost)

func execute_forge(card: Dictionary, forge_type: String, params: Dictionary = {}) -> Dictionary:
    var forged = card.duplicate(true)
    
    match forge_type:
        "emotion_brand":
            forged = _apply_emotion_brand(forged, params.get("emotion_type", "bei"))
        "mechanism_link":
            forged = _apply_mechanism_link(forged, params.get("link_type", "seal"))
        "resonance_amp":
            forged = _apply_resonance_amplify(forged)
        "chain_enhance":
            forged = _apply_chain_enhance(forged)
        "emotion_swap":
            forged = _apply_emotion_swap(forged, params.get("from"), params.get("to"))
        "soul_link":
            forged = _apply_soul_link(forged)
        "void_inject":
            forged = _apply_void_inject(forged)
        "ultimate_recast":
            forged = _apply_ultimate_recast(forged)
    
    forged["forged"] = true
    forged["forge_type"] = forge_type
    forged["display_name"] = forged.get("name", "") + " 【锻】"
    
    # 消耗碎片
    var cost = FORGE_TYPES[forge_type]["shard_cost"]
    DiscardSystem.consume_shards(cost)
    
    DeckManager.replace_card(card.get("id",""), forged)
    emit_signal("forge_completed", card, forged)
    return forged

func _apply_emotion_brand(card: Dictionary, emotion_type: String) -> Dictionary:
    card["on_play_extra"] = card.get("on_play_extra", [])
    card["on_play_extra"].append({
        "type": "add_emotion",
        "emotion": emotion_type,
        "amount": 1
    })
    return card

func _apply_chain_enhance(card: Dictionary) -> Dictionary:
    card["forge_chain_bonus"] = card.get("forge_chain_bonus", 0) + 5  # +5%溅射
    return card

func check_and_unlock_recipes() -> void:
    # 检查图纸解锁条件
    var purification_count = GameState.total_purification_count
    var rage_burst_count = GameState.total_rage_burst_count
    
    if purification_count >= 3 and "du_hua_recipe" not in unlocked_recipes:
        unlocked_recipes.append("du_hua_recipe")
        emit_signal("recipe_unlocked", "du_hua_recipe")
    
    if rage_burst_count >= 5 and "rage_forge_recipe" not in unlocked_recipes:
        unlocked_recipes.append("rage_forge_recipe")
        emit_signal("recipe_unlocked", "rage_forge_recipe")
```

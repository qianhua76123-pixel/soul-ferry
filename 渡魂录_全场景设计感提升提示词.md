# 《渡魂录》全场景 UI 设计感提升提示词

> 基于四张截图的系统评审。当前最大问题不是某个场景，
> 而是**四个场景之间没有共同的设计语言**——像四个独立的原型拼在一起。
> 本文档分两层：先建立全局设计系统，再逐场景强化。

---

## 📊 现状速览

| 场景 | 已做到的 | 最需要改的 |
|------|---------|----------|
| 主菜单 | 标题字体有气质，红色暗角氛围好 | 按钮没有个性，背景是死黑 |
| 地图 | 节点颜色区分清晰 | 纯黑背景无场景感，连接线是破折号，节点无图标 |
| 商店 | 烛台图标有味道，副标题文案好 | 卡牌太小，下半部分大片空白 |
| 战斗 | 能量格(|||)很出彩，五情进度条清晰 | HP条灰色无信息，左右两侧大面积死区 |

## 模块 DS-00：全局设计系统（必须先建立）

```
【任务】为《渡魂录》建立统一的视觉设计系统，以 GDScript 常量文件的形式输出，
供所有场景的 UI 节点引用。

【游戏视觉定位】
民俗志怪 · 像素风 · 水墨笔触
参考气质：《鬼谷八荒》的界面厚重感 + 像素艺术的克制 + 中国水墨的留白哲学
核心矛盾：像素立绘的颗粒感 vs 界面需要的精致感 → 用"做旧纸张质感"统一两者

【色彩体系】
主色调命名为"渡魂五色"，对应五情：

const COLORS = {
    # 基础色
    "ink":      Color("#0d0d0d"),   # 墨黑——背景底色
    "parch":    Color("#1a1508"),   # 泛黄纸色——面板底色
    "gold":     Color("#c8a96e"),   # 旧金——主要强调色
    "gold_dim": Color("#6b5a30"),   # 暗金——次要线条
    "ash":      Color("#9a9080"),   # 灰烬——禁用/次要文字

    # 五情色（与 EmotionManager 对应）
    "nu":    Color("#c0392b"),   # 怒——朱砂红
    "ju":    Color("#6c3483"),   # 惧——暗紫
    "bei":   Color("#1a5276"),   # 悲——深青
    "xi":    Color("#b7770d"),   # 喜——琥珀金
    "ding":  Color("#1d6a54"),   # 定——松绿

    # 节点类型色（地图用）
    "battle": Color("#7b1a1a"),  # 战斗——深血红
    "shop":   Color("#5a4a00"),  # 商店——旧金
    "event":  Color("#1a3a1a"),  # 事件——墨绿
    "rest":   Color("#1a2a3a"),  # 休息——深蓝
    "boss":   Color("#3d0000"),  # Boss——极暗红
}

【字体层级】
const FONT_SIZES = {
    "title":   32,  # 场景标题（渡魂商店、五情祭坛）
    "heading": 18,  # 区块标题
    "body":    13,  # 正文
    "caption": 11,  # 辅助信息
    "micro":   10,  # 图标标注
}

【边框/面板语言】
所有面板使用三层结构（从外到内）：
1. 外层暗边（1px，颜色 gold_dim，透明度 0.4）
2. 内层填充（颜色 parch，透明度 0.88）
3. 顶部装饰线（2px，颜色 gold，只绘制顶边）

面板四角：使用小三角切角（非圆角）——参考印章/古籍的方正感
GDScript 用 _draw() + draw_line 实现切角矩形：
func draw_inked_panel(rect: Rect2, color_fill: Color, color_border: Color):
    # 切角大小 = 6px
    var c = 6.0
    var p = rect
    var points = PackedVector2Array([
        Vector2(p.position.x + c, p.position.y),
        Vector2(p.end.x - c,      p.position.y),
        Vector2(p.end.x,          p.position.y + c),
        Vector2(p.end.x,          p.end.y - c),
        Vector2(p.end.x - c,      p.end.y),
        Vector2(p.position.x + c, p.end.y),
        Vector2(p.position.x,     p.end.y - c),
        Vector2(p.position.x,     p.position.y + c),
    ])
    draw_colored_polygon(points, color_fill)
    draw_polyline(points + PackedVector2Array([points[0]]), color_border, 1.0)

【分割线样式】
不用直线，改用"水墨渗透"分割线：
- 中间最深（透明度 0.6），向两端渐淡（透明度 0）
- 用3条平行线叠加实现（宽度 1px/0.5px/0.5px，透明度递减）
- 颜色统一用 gold_dim

【图标系统】
所有 UI 图标统一用像素字体 Unicode 字符，不依赖外部资源：
- 金币：🪙 → 改为手绘1像素铜钱图标（6×6 ImageTexture，代码生成）
- 护甲：🛡 → 4×5 像素六边形盾
- 生命：♥ → 5×5 像素心形
- 能量：▮ → 已实现，保留
节点类型图标（地图用）：
- 战斗：交叉剑纹（✕形，10×10px）
- 商店：算盘/铜钱（○形，8×8px）
- 事件：竹简卷（▤形，8×10px）
- 休息：篝火（△形，8×10px）
- Boss：鬼面（◉形，10×10px）

【输出】
1. UIConstants.gd 完整文件（颜色/字号/字符常量）
2. InkedPanel.gd（可复用的切角面板绘制组件）
3. WaterInkDivider.gd（水墨分割线组件）
4. 所有场景 .gd 文件中引入常量的方式
```

## 模块 M-01：主菜单——让按钮有仪式感

```
【当前问题】
对照截图：按钮是三个相同的灰色矩形，hover 效果不明显，
与游戏标题的气质落差极大——标题有分量，按钮像调试界面。

【目标效果参考】
想象三根竖立的竹简，每根刻着一行字，悬挂在标题下方。
或：三块印章，盖在泛黄的宣纸上。

【改版规格】

1. 背景层升级
当前：纯黑 + 红色暗角（效果已不错，保留红色暗角）
新增：
  a. 水墨竖向纹理层（BattleBackground 里已写过的代码，复用）
     用固定 seed 的随机竖线，透明度极低（0.04）
  b. 画面中央下方淡淡的"香炉烟雾"粒子（GPUParticles2D）：
     粒子形态：细长矩形 1×4px，颜色 #c8a96e（旧金），透明度 0.06-0.12
     向上漂散，speed 30px/s，spread 20°，从画面底部中心发射

2. 按钮重设计（三个按钮各有差异感）

按钮基础样式（StyleBoxFlat 弃用，改用自定义 draw）：
  - 宽度 220px，高度 48px
  - 切角矩形（使用 InkedPanel 组件）
  - 静止状态：fill = Color("#1a1508", 0.7)，border = gold_dim
  - 文字：白色，字号16，字间距 +4px（汉字拉宽间距）

"踏上旅途"——主要操作：
  - 边框颜色使用 gold（亮金），顶部装饰线存在
  - hover：整体向上偏移 2px + 边框加亮
  - 前置装饰：左侧加一个小"✦"符号，颜色 gold

"继续渡魂"——次要操作：
  - 无存档时：整体透明度 0.4，文字变 ash 色，"✦"替换为"—"
  - 有存档时：与主按钮类似但边框用 gold_dim

"就此搁笔"——危险操作：
  - 文字颜色 ash（不强调），无前置符号
  - hover：文字变 Color("#c0392b")（朱砂红），无其他动画

3. 标题区增加进场动画
游戏启动时：
  a. "渡魂录"三字从透明淡入（0.8s），同时从 y+20 归位
  b. 0.5s 后副标题淡入（0.4s）
  c. 1.0s 后分割线从中间向两侧展开（0.3s）
  d. 1.3s 后按钮逐个淡入（间隔 0.15s）
全部用 Tween 链式调用实现。

4. 版本号升级
"v0.1 Demo" → 右下角，字号10，颜色 gold_dim，加前缀"✦ "

【输出】
1. MainMenu.gd 的 _ready() 进场动画完整 Tween 链
2. MenuButton.gd 自定义按钮组件（含三种状态的 _draw 实现）
3. 背景香炉粒子的 GPUParticles2D 参数配置
```

## 模块 M-02：地图场景——让旅途有重量感

```
【当前问题】
对照截图：三层节点排列在纯黑背景上，连接线是"—"破折号，
节点内容是"[战]战斗"文本标签，感觉像电子表格而非冒险地图。
第二、三层灰色节点和第一层几乎一样，已走和未走没有直观区别。

【目标效果】
想象一张用毛笔画在旧宣纸上的路途图——
荒村的节点带着锈蚀感，古祠的节点更厚重，幽冥渡口的节点半透明如幻。
玩家走过的节点留下"到此一游"的痕迹。

【改版规格】

1. 背景分层（三层各有氛围色）
用 ScrollContainer 包裹三个 SubViewport，各层有独立背景：
  - 第一层（荒村）：背景主色 #0a1008（深墨绿），暗示江边植被
  - 第二层（古祠）：背景主色 #120a08（暗棕），暗示古旧木石
  - 第三层（幽冥渡口）：背景主色 #08080f（深夜蓝），暗示幽冥
每层背景：叠加 BattleBackground 的水墨竖线，透明度 0.03-0.06

2. 节点图标替换文字标签
当前节点显示"[战]\n战斗"，改为图标+小字：
节点尺寸 64×64px（比当前大），用 _draw() 实现：
  
  战斗节点：深血红底色 + 白色"✕"符号（交叉剑意象）+ 底部微小"战"字
  商店节点：旧金底色 + 白色"◎"符号（铜钱意象）+ "店"字
  事件节点：墨绿底色 + 白色"▤"符号（竹简意象）+ "事"字
  休息节点：深蓝底色 + 白色"△"符号（篝火意象）+ "息"字
  Boss节点：极暗红底色 + 特殊边框（双层，外层发光效果）+ "魔"字
  
  所有节点使用 InkedPanel 的切角矩形，不用圆角。

3. 连接线升级
当前：文字"—"
改为：draw_line() 绘制的虚线路径
  - 已走过的路径：颜色 gold_dim，不透明，实线，宽1.5px
  - 未走路径：颜色 ash，透明度 0.3，虚线（dash 6px，gap 4px），宽1px
  - 路径中间画一个极小的方向符号（▶，5×5px，颜色与路径一致）

4. 节点状态系统
  - 已完成：节点底色变暗（乘以 0.5），中心画一个"✦"印记，颜色 gold_dim
  - 当前可选：轻微脉冲发光（透明度 0.8→1.0→0.8，1.5s 循环），边框用 gold
  - 已锁定（下层）：整体透明度 0.25，不可点击，鼠标变禁止光标
  - 鼠标悬停：向上偏移 3px + 边框颜色加亮，显示 Tooltip

5. 层间过渡
三层之间的过渡箭头（当前是"▼"）改为：
  - 一段"向下的路"图示：10px 宽的路径，从第一层底部向第二层延伸
  - 路径样式与节点连接线相同（虚线，颜色 ash）
  - 旁边文字："第N层 · 层名（情绪）"

6. 玩家位置指示
当前层的某个节点旁加一个"渡魂人"小像素图（复用 BattleScene 的立绘，
缩小到 20×20px），站在当前节点下方。

7. 顶部 HUD 升级
当前："渡魂录 SOUL FERRY     HP: 80/80  🪙 100"
改为：
左：渡魂录 · 第N层 荒村  
右：♥ 80/80  [血条小图，50px宽]  ◎ 100
遗物栏：移到右侧顶部下方，横向排列，遗物有 hover tooltip

【输出】
1. MapNode.gd 完整重写（_draw 实现切角图标，三种状态动画）
2. MapPath.gd（路径绘制组件，处理虚线/实线切换）
3. MapScene.gd 中节点状态管理的修改代码
4. 各层背景色在 _ready() 中根据层号动态设置的代码
```

## 模块 M-04：战斗场景——消灭死区，强化焦点

```
【当前改进进展评价（对照新截图）】
✅ 已做好：能量格(|||)有设计感，五情进度条清晰，渡化条件面板已集成
⚠ 仍需改：HP条纯灰色，玩家/敌人两侧死区，敌人意图是占位符

【问题1：两侧死区】
当前：玩家立绘+状态在左侧 240px 区域，右侧 240px 是敌人，
中间 700px 是五情祭坛，但祭坛容器只占约 400px，两侧各有 150px 空白。

解决方案：
将玩家区和敌人区"靠拢"战场：
  - 玩家区：立绘底部对齐"战场地面线"（y = 战场区 65%），
    立绘左侧距屏幕边缘保持 20px，右侧放状态栏
  - 敌人区：同理，向左靠近至距五情祭坛容器右边缘 20px
  - 五情祭坛容器宽度从 400px 扩展到屏幕宽度 50%（约 600px）

【问题2：HP条设计感不足】
当前：纯灰色进度条 + "100%" + "80/80"

改版规格：
玩家 HP 条（宽 200px，高 14px）：
  结构：[♥ 图标 8px] [血条] [数值 "80/80"]
  血条样式（用 _draw 实现，不用默认 ProgressBar 样式）：
    - 底层：深色槽（#1a0808，高14px，切角4px）
    - 实际HP层：根据比例变色
      > 60%：#4a7a3a（松绿）
      30-60%：#8b6914（暗金）
      < 30%：#8b1a1a（血红）+ 每1.5s脉冲闪烁
    - 顶部1px亮边（颜色比填充色亮20%）
    - 右侧文字：字号11，颜色与填充色一致

敌人 HP 条（宽 160px，高 12px）：
  同规格，颜色体系反向（用于区分玩家/敌人）
  底部追加意图预告（见下方）

【问题3：敌人意图】
当前："？  蓄势待发"

改版规格（参见之前的 B-04 提示词，这里补充视觉实现）：
意图区域（位于敌人HP条正下方）：
  - 背景：InkedPanel，宽160px，高32px
  - 图标（12×12px Unicode字符）+ 行为描述（字号11）
  - 整体在每次意图切换时有 0.3s 淡入动画
  - 攻击意图时：图标做 ±3px 横向轻微摇晃（0.6s 循环，暗示威胁感）

【问题4：五情祭坛标题重复】
"—— 五情祭坛 ——"在雷达外部有，雷达内部黄色框上又出现，重复了。
移除内部的"五情祭坛"文字标签，保留外部标题即可。

## 模块 M-05：场景过渡动画

```
【任务】为场景切换添加统一的过渡动画，增强游戏的整体感。

【当前状态】
场景切换是瞬间跳变，没有任何过渡效果。

【设计方案：水墨晕散过渡】

所有场景切换统一使用以下序列（总时长 0.8s）：
  Out 阶段（0.4s）：
    - 从屏幕中心开始，黑色 ColorRect 扩展铺满整个屏幕
    - 扩展方式：radius 从 0 增长到 screen_diagonal（用 shader 实现圆形遮罩）
    - 扩展过程中叠加水墨扩散感（边缘不整齐，用 noise 扰动）
  
  In 阶段（0.4s）：
    - 新场景从中心向外展开（反向）
    - 在圆形完全消失前，新场景已开始显示

【Godot 实现方式】
使用 CanvasLayer（z_index=100）+ ColorRect + ShaderMaterial：

```glsl
// transition.gdshader
shader_type canvas_item;
uniform float progress: hint_range(0.0, 1.0) = 0.0;
uniform vec2 center = vec2(0.5, 0.5);
uniform sampler2D noise_tex;

void fragment() {
    vec2 uv = SCREEN_UV;
    float dist = distance(uv, center);
    float noise = texture(noise_tex, uv * 3.0).r * 0.08;
    float mask = step(progress + noise, dist);
    COLOR = vec4(0.05, 0.04, 0.02, mask);
}
```

TransitionManager.gd（Autoload）：
```gdscript
func transition_to(scene_path: String):
    _overlay.visible = true
    var tween = create_tween()
    # Out
    tween.tween_property(_shader_material, "shader_parameter/progress", 
                         0.0, 0.4).set_ease(Tween.EASE_IN)
    tween.tween_callback(get_tree().change_scene_to_file.bind(scene_path))
    # In  
    tween.tween_property(_shader_material, "shader_parameter/progress",
                         1.0, 0.4).set_ease(Tween.EASE_OUT)
    tween.tween_callback(_overlay.hide)
```

把所有 get_tree().change_scene_to_file() 调用替换为 
TransitionManager.transition_to(path)

【特殊过渡：进入战斗】
进入战斗场景时，额外在过渡后播放一个 0.3s 的"战鼓震动"屏幕抖动
（整个场景 CanvasLayer 做 ±4px 横向偏移，3次，0.1s间隔）

【输出】
1. transition.gdshader 完整代码
2. TransitionManager.gd 完整代码
3. 替换现有场景切换调用的清单和代码
4. 进入战斗的特殊抖动效果实现
```

## 模块 M-06：事件场景——叙事氛围与选择可读性

```
【任务】把“事件”做成能阅读的短篇，而不是弹窗堆字。重点是：信息密度、选择按钮的仪式感、以及水墨留白节奏。

【当前问题（通常会出现）】
- 背景纯色或纹理过弱，缺少“发生过”的感觉

【改版规格】
1. 事件面板
   - 使用 InkedPanel 的三层面板语言（切角矩形 + 顶部装饰线）
   - 面板填充用 parch，外层暗边 gold_dim（透明度 0.4）
   - 在面板背后加一层轻微纸张噪声（透明度 0.04~0.06 的水墨颗粒）

2. 叙事文本排版
   - 标题用 FONT_SIZES.heading（18）+ 金色 gold_dim 做小字描边感（draw text shadow 或额外 1px 渲染）
   - 正文字号 13，行距 1.25（用 RichTextLabel 自定义行距）
   - 每段之间用 WaterInkDivider 的“水墨渗透”分割线替代空行

3. 选择按钮（复用菜单按钮的“仪式感”，但做小型化）
   - 选择区域底部固定高度 120px
   - 每个按钮宽度 280px，高度 44px
   - 按钮文字字间距 +3px
   - 左侧加 1 个选择指示符：
     - 正向收益：gold 色 “✦”
     - 不确定/代价：ash 色 “—”
     - 负向：朱砂红 “✕”
   - hover：整体上浮 2px + 边框 gold 亮边

4. 背景升级（事件专属）
   - 在底部 1/3 区域叠加“竹简阴影条”（窄条渐变矩形，透明度 0.05~0.1）
   - 若该事件来自地图节点类型 event，则在背景左上角放一个 8x10px 的“▤ 竹简意象”小标记

5. 动画节奏（短促，不要拖）
   - 面板入场：0.25s 淡入 + 0.15s 从 y+10 回落
   - 正文落字：0.35s 内完成（用 Tween 修改 alpha）
   - 选择按钮：在正文结束后，每个按钮间隔 0.08s 淡入

【输出】
1. EventScene.tscn 的节点树：面板/正文/选择区明确拆分
2. EventScene.gd：rich text 排版与按钮状态机
3. 选择按钮组件（可复用 MenuButton 的 draw 逻辑，参数化尺寸）

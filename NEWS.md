## 已完成功能总览

### ✅ 核心系统（Autoload，共8个）

| 顺序 | 名称 | 功能 |
|------|------|------|
| 1 | `ThemeSetup` | 全局朱红暗金像素风主题，覆盖8种控件样式 |
| 2 | `EmotionManager` | 五情系统（怒/惧/悲/喜/定），情绪变化信号，失调判断，战斗修正倍率 |
| 3 | `CardDatabase` | JSON 牌库加载（28张牌），动态描述（BBCode 升级高亮） |
| 4 | `DeckManager` | 牌库/手牌/弃牌堆管理，洗牌，摸牌，出牌，费用控制 |
| 5 | `GameState` | 游戏全局状态（HP/金币/层数/遗物/记录），存档系统，结局触发 |
| 6 | `RelicManager` | 10个遗物完整效果，触发信号，问路香/年画眼等特殊逻辑 |
| 7 | `BuffManager` | 4种 Buff（灼烧/中毒/执念/护盾），回合处理，图标栏 Tooltip |
| 8 | `SoundManager` | 28种 SFX + 13种 BGM（占位符架构），3路并发播放池，场景自动切 BGM |

### ✅ 战斗系统

#### 战斗状态机（`BattleStateMachine.gd`）
- 7个状态：IDLE / BATTLE_START / PLAYER_TURN / RESOLVING / ENEMY_TURN / TURN_END / BATTLE_END
- 整数常量替代 enum，避免外部引用报错
- 卡牌效果处理：20种 effect_type（攻击/护盾/回血/摸牌/弱化/渡化触发等）
- 敌人 AI：加权随机行动选择，支持 dot/情绪干扰/护盾/召唤等行动类型
- 渡化系统：条件判断（情绪要求+指定牌/连续喜牌），触发渡化结局

#### 战斗场景（`BattleScene.gd`）
- 祭坛式三栏布局（玩家区｜五情雷达｜敌人区）
- 迷你遗物栏（战斗中显示，支持 Tween 闪光动画）
- Buff 图标栏（双侧，鼠标悬停 Tooltip）
- 浮字系统（受伤/回血/护盾/情绪变化，各有颜色和位置）
- 渡化按钮（条件满足时出现，带提示文字）
- 问路香按钮（持有时动态添加到 HUD）

#### Boss 专属 UI（`BossUI.gd`）
- 入场动画（透明+放大→原尺寸，0.6s 渐入+抖动）
- 血条颜色随 HP 变化（深红→橙红→金黄垂死）
- 阶段系统：HP ≤ 50% 触发愤怒阶段（全屏红闪+立绘常驻脉冲）
- 意图预告（图标+文字，愤怒阶段标记）
- BOSS 标签金色脉冲动画

### ✅ 存档系统

- 自动存档：地图节点点击后触发
- 保存内容：HP / 金币 / 牌库（含升级等级）/ 遗物 / 情绪值 / 地图状态 / 路线倾向 / 渡化/镇压记录
- 存档路径：`user://save.json`（Windows: `%APPDATA%\Godot\app_userdata\soul_ferry\`）
- 通关/失败后自动清档

## 当前游戏完整流程

```
主菜单（MainMenu）
  ├─ 新游戏 → 地图（第1层）
  └─ 继续渡魂 → 地图（读取存档）

地图（MapScene）
  ├─ 战斗节点 → 战斗场景 → 选牌奖励 → 地图
  ├─ 事件节点 → 事件场景 → 地图
  ├─ 商店节点 → 商店场景 → 地图
  ├─ 休息节点 → 回复HP → 地图
  └─ Boss节点 → 战斗场景（Boss模式）
                  ├─ 胜利/渡化 → 选牌奖励 → 下一层地图
                  └─ 通关第三层 → 结局场景（渡魂成功）

战斗失败 → 结局场景（魂魄消散）
特殊路线 → 结局场景（迷失轮回·隐藏）
```
